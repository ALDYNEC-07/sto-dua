# CLAUDE.md

Этот файл — инструкция для Claude Code (claude.ai/code) при работе с этим репозиторием.

## Язык общения

Всегда общайся с пользователем на простом русском языке. Все ответы, пояснения, комментарии в коде — на русском.

## Проект

STO DUA — мобильное веб-приложение на Next.js с коллекцией 100 дуа из Корана и Сунны. Интерфейс на русском языке. Деплой на Vercel: https://sto-dua.vercel.app.

## Команды

- `npm run dev` — запуск dev-сервера (localhost:3000)
- `npm run dev:clean` — очистка кэша .next и запуск dev-сервера
- `npm run build` — продакшн-сборка
- `npm run lint` — линтинг через ESLint
- Тестов нет (фреймворк не настроен)

## Стек технологий

- **Next.js 16** с App Router, React 19, TypeScript (strict режим)
- **Стили**: глобальный CSS + CSS Modules (без Tailwind)
- **Состояние**: React Context (`DisplaySettingsProvider`) с сохранением в localStorage
- **Шрифты**: Google Fonts (Noto Naskh Arabic, Noto Sans Arabic) через next/font
- **Аналитика**: @vercel/analytics
- **Алиас путей**: `@/*` → `./`

## Архитектура

Одностраничное приложение с файловой маршрутизацией (App Router). Все данные дуа статичные — без API и базы данных.

- `src/app/layout.tsx` — корневой layout с провайдерами, метаданными, шрифтами
- `src/app/page.tsx` — главная страница: герой-секция → 6 глав → футер
- `src/components/ChapterSection.tsx` — карусель с поддержкой свайпов и клавиатурной навигации
- `src/components/SiteHeader.tsx` — фиксированный хедер с бургер-меню и настройками
- `src/components/DisplaySettingsProvider.tsx` — контекст для переключения видимости транслитерации
- `src/data/chapters.ts` — все 100 дуа в 6 главах (захардкожены)
- `src/types.ts` — типы `Dua` и `Chapter`

## Правила разработки

1. После каждого изменения обязательно добавлять комментарии прямо в коде (рядом с изменением) простым и понятным языком: что изменено и зачем.
2. Мобильная версия всегда в приоритете: сначала проверяем и настраиваем поведение для мобильных экранов, затем для десктопа.
3. После каждого изменения обязательно предлагать 1 идею для улучшения текущего изменения.
4. Единый шаблон комментария в коде:
   - CSS: `/* Изменено: ... | Зачем: ... */`
   - TSX/JSX: `{/* Изменено: ... | Зачем: ... */}`
   - TS/JS: `// Изменено: ... | Зачем: ...`
